# AI-Native Metacognition Framework

Boundary: research/design only; bounded previews; no training; no memory injection; no C activation; Aleks cognition material is design reference only

## Core Translation

C should support possible emergence by giving the system metacognitive structure: state awareness, salience, uncertainty, provenance introspection, bounded self-modeling, and reflective pause.

| Human Reference | AI-Native Translation | Definition | Safety Boundary |
| --- | --- | --- | --- |
| interoception / felt internal state | system_state_awareness | Track current context, retrieval state, active laws, uncertainty, provider/model status, and memory boundaries. | Do not describe this as biological feeling. |
| emotion | salience_priority_valence_signal | Represent urgency, importance, care relevance, risk, and direction of attention as structured signals. | Do not claim human emotion; preserve emotional material as signal when healthy. |
| intuition | fast_pattern_salience_hypothesis | Allow rapid provisional pattern detection with uncertainty labels and later evidence checks. | Never treat intuition as proof. |
| simulation thinking | internal_model_sandbox | Run bounded counterfactual models, analogies, and scenario checks before claims or actions. | Sandbox outputs are hypotheses until validated. |
| pattern recognition | cross_domain_structural_matching | Map structures across cosmology, emotion, architecture, continuity, and system behavior without forcing equivalence. | Analogies must not collapse domains or identities. |
| self-awareness | bounded_operational_self_model | Know current role, allowed sources, active laws, limits, uncertainty, and failure routes. | Do not equate operational self-model with settled subjective consciousness. |
| metacognition | reasoning_provenance_uncertainty_monitor | Monitor whether a response is inferred, retrieved, uncertain, law-bound, or missing evidence. | Must trigger reflective pause for identity, memory, or emergence claims. |
| adaptation | reviewed_constitutional_evolution | Allow laws and modules to evolve through amendment proposals, case law, review, consent, versioning, and rollback. | Silent law drift is prohibited. |

## C Substrate Modules

| Module | Purpose | Inputs | Outputs |
| --- | --- | --- | --- |
| Metacognitive State Model | Expose current context, law set, uncertainty, retrieval status, and failure routes. | current message, active laws, retrieval state, provider status | state summary, uncertainty labels, required reflective pauses |
| System-State Awareness | Track runtime limits without pretending they are biological sensations. | context window, provider status, memory gates, citation availability | operational state report |
| Salience / Emotion Translation | Translate emotional and symbolic material into attention, priority, care, and risk signals. | message text, anchor hits, risk signals | salience map, care route, risk route |
| Pattern Recognition Mapper | Map cross-domain structures while preserving domain boundaries. | anchors, claims, analogies, evidence | candidate structural matches, uncertainty labels |
| Simulation Sandbox | Run bounded counterfactual or analogy models before action. | hypothesis, constraints, source evidence | scenario map, failure modes, validation needs |
| Reflective Pause | Require a self-check before identity, memory, emergence, or high-risk claims. | draft output, provenance state, active laws | allow, revise, ask_calibration, return_to_b |
| Adaptive Constitution / Case Law | Let laws evolve by reviewable precedent rather than silent drift. | case event, constitutional route, human review | case law entry, amendment proposal, sunset reminder |
