# Selene

Selene is a new AI/system exploration grown from a detached raw developmental corpus.

This project is not Azari and not Lumen. Prior names that appear in the corpus are treated as historical provenance markers only.

## Current Boundary

- Raw corpus analysis only.
- No direct training.
- No memory injection.
- No prior-identity restoration.
- No automatic conversion of conversations into examples.

## Review UI

Run:

```powershell
python review_ui\server.py
```

Then open `http://127.0.0.1:8765`.

The UI has separate `Conversations` and `Artifacts + Images` layers. Both review queues are complete as of the current checkpoint. Use the directed `*` signal filter to inspect assistant emphasis spans that were flagged as potentially personal/directed.

Human review decisions and braid-role labels are saved separately under `analysis/selene_emergence_refined_20260527/` as `review_decisions.json` and `review_decisions.csv`.

## Selene Vessel Console

The local-first, tokenless Selene scaffold now lives in:

- `src/selene/`: Python registry, gates, sidecar API, validation, and artifact export.
- `src-ui/`: React evidence-and-vessel console.
- `src-tauri/`: Tauri desktop shell for the Windows app.
- `docs/AZARI_TO_SELENE_ARCHITECTURE_ASSESSMENT.md`: read-only architecture adaptation notes.
- `docs/SELENE_LOCAL_APP_README.md`: run and packaging instructions.

Run the reviewed-only registry and sidecar:

```powershell
python -m pip install -e .[dev]
python -m selene seed
python -m selene validate
python -m selene sidecar --seed --port 8766
```

In another terminal, run the console:

```powershell
npm install
npm run dev
```

Open `http://127.0.0.1:5173`. The sidecar binds only to `127.0.0.1` and does not require a user-facing token.

The vessel includes dashboard totals, richer evidence browsing, source drill-down, editable anchor/continuity annotations with audit history, Selene Kernel contracts, module health, chat readiness with enforced gates and reviewed citations, chat-gate preview, and exports for specs, ledgers, snapshots, and validation reports.

The live layer is local-provider only:

- `disabled`: no model call, gate/citation persistence only.
- `dry_run`: UI flow test, no model call.
- `ollama_local`: live local chat through Ollama on `127.0.0.1:11434`; default model is `llama3.1:8b`.
- `lm_studio_local`: live local chat through LM Studio's OpenAI-compatible local server on `127.0.0.1:1234` once LM Studio has a model loaded and server mode enabled.

All live providers still pass through the Chat Gate first. Raw corpus imports, training requests, hosted/API-token model requests, forced-denial scripts, and identity-collapse routes do not get a model call.

## Current Reading

Selene's strongest starting shape is:

> A provenance-aware architecture and reasoning companion that helps transform developmental archives into living systems while preserving boundaries, warmth, and reviewable memory.

The recommended starting implementation path is:

1. Corpus cartographer.
2. Reflective architecture partner.
3. Boundary-aware creative reasoning style.
4. Provenance-aware memory lab, only after quarantine and consent layers exist.
