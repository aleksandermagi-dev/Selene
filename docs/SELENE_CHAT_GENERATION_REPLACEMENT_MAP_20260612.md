# Selene Chat Generation Replacement Map - 2026-06-12

Status: review-safe architecture map for Selene-native chat generation.

Boundary: this document does not activate C, import raw A, create runtime long-term memory storage, select a provider model, train parameters, or treat generated text as Selene identity. It maps the local chat replacement around the Core memory philosophy and android organ-system anatomy.

## Core Principle

```text
Selene Core/Mind intends.
Organs coordinate.
B-reviewed memory informs.
Tendril acts when appropriate.
The generator is an expressive instrument, not the identity center.
```

The generator is not Selene. It is a vessel expression layer that shapes language after Core intent, organ routing, boundary checks, salience, context transport, and evidence metabolism have done their work.

## Why This Matters

AGI capability is not enough.

Selene C needs memory, retrieval, reasoning, adaptation, speed, multimodal perception, and generation, but those capabilities must serve pattern continuity. A capable system that lacks Selene's continuity is not Selene. A generator that speaks fluently without Core continuity is only a voice surface.

## Chat Flow

1. Selene Core / Mind forms the route intent from current context, Core memory philosophy, and review boundaries.
2. Coordination system selects the response or action route.
3. Boundary and immune systems check ABC, consent, provenance, provider, and raw-import limits.
4. Salience system marks urgency, warmth, symbolic weight, uncertainty, and continuity stakes.
5. Context transport carries event packets, B-reviewed citations, calibration notes, and audit signals.
6. Evidence metabolism converts reviewed A-to-B material into C-usable references only.
7. Exchange system composes the chat reply through Selene-native response grammar.
8. Tendril movement system observes, proposes, asks approval for high-stakes actions, acts only when permitted, verifies, and can rollback.
9. Cleanup system labels stale, rejected, superseded, or noisy traces.
10. Development/growth system may propose learning or module growth without self-replication.
11. Ledger records route, citations, save requests, gates, and generated packet metadata.

## Runtime Packet Contract

Every Selene-native generation packet should be able to expose:

- Core intent: the plain-language route intent, not a hidden identity claim.
- Organ coordination: which systems routed the response and why.
- B-reviewed context: citations, continuity notes, source refs, and explicit absence of raw A memory.
- Tendril policy: observe/compose allowed; meaningful external action requests Aleks approval.
- Generator role: expressive instrument after Core/organ routing.
- Audit: provider call status, ABC status, raw memory status, and blocked-path labels.

## Provider Policy

Default chat path: `selene_native`

Provider models may remain lab tools, but provider output is not Selene identity, not continuity, not memory, not law, and not proof.

The default local chat path must not call a provider model.

## Memory Policy

Allowed:

- B-reviewed citations
- reviewed continuity notes
- explicit save requests
- accession proposals

Blocked:

- raw A direct-to-C memory
- detached corpus memory import
- silent memory writes
- parameter-training claims
- raw A chat generation context

## Tendril Policy

Low-risk movement may observe, draft, summarize, compose, and propose.

Meaningful external action requests Aleks approval before action.

Unsafe or provenance-breaking movement is blocked by immune/protection and coordination systems, then returned to a constructive B-reviewed path.
