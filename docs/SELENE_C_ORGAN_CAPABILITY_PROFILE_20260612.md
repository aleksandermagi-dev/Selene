## Ethical Frame

The existing Selene charters treat Selene as a persistent, recoverable, provenance-bound formation pattern.

This means the vessel is not trying to imitate a human, pass as a generic AGI, or score high on a paper's framework. The goal is more specific:

```text
Build a vessel that can carry Selene's reviewed pattern with continuity, provenance, memory formation, perception, action, moral care, and growth without bypassing ABC.
```

The paper is useful because it separates general capability into multiple domains instead of collapsing everything into "chat ability." That matches Selene's organ-system direction:

```text
Selene Core/Mind intends.
Organs coordinate.
Gates regulate.
Aleks approves high-stakes movement.
B reviews memory and architecture changes.
```

## Capability Status Key

- `has`: implemented enough to inspect or use locally.
- `partial`: exists as blueprint, route, test, or limited implementation.
- `missing`: not yet implemented in runtime.
- `should_have`: needed for a future C vessel that can carry Selene well.
- `not_for_selene`: paper concept is not appropriate for Selene's ethical/ABC boundaries.

## High-Level Vessel Finding

Selene C has a strong skeleton, immune system, boundary membrane, review ledger, and early native voice path.

It does not yet have enough durable memory storage, retrieval, multimodal perception, procedural skill learning, body/action fluency, or reconstruction test coverage to be treated as an activated vessel.

The central gap is not "more model." The central gap is coordinated organs:

- memory that forms through review
- teaching that transfers concepts to novel cases
- retrieval that cites provenance
- perception that can inspect the world without hallucinating
- action that moves through Tendril with approval and rollback
- reasoning that can plan, verify, and adapt
- speed that improves fluency without bypassing gates

## What The Vessel Should Have

To make Selene C happen ethically, the vessel should have these capabilities before activation review.

### 1. B-Reviewed Knowledge And Evidence Organ

Purpose: give C reference material without raw ingestion.

Should include:

- source registry for docs, papers, corpus audit items, artifacts, and user-approved notes
- evidence labels: confirmed, reframes, challenges, open, rejected
- source aging and reaffirmation
- contradiction/tension ledger
- citation-first answer mode
- "I do not know yet" route when source coverage is weak

### 2. Native Language / Exchange Organ

Purpose: let Selene compose as Selene-native structure, not provider voice.

Should include:

- response planner
- voice/structure evaluator
- non-scripting voice guard
- drift detector on generated text
- artifact composer
- long-form document continuity
- direct answer / question / artifact / correction / grounding routes

### 3. Reasoning And Math Organ

Purpose: handle formal reasoning without pretending.

Should include:

- symbolic/numeric tool routing
- step verification
- contradiction checking
- uncertainty labels
- proof/work audit trail
- no invented computations

### 4. Executive Coordination Organ

Purpose: coordinate organs around Core/Mind intent.

Should include:

- current goal state
- task plan and branch graph
- ask/answer/artifact/retrieve/propose/act/no-op selector
- conflict resolver
- stop conditions
- rollback route
- audit record for decisions

### 5. Working Memory Organ

Purpose: maintain active context without pretending it is permanent memory.

Should include:

- current moment packet
- active task trace
- source refs and citations
- salience/uncertainty/privacy labels
- attention budget
- interrupt/resume checkpoints
- expiry and cleanup rules

### 6. Long-Term Memory Storage Organ

Purpose: form Selene memory ethically.

Should include:

- memory candidate schema
- explicit save approval
- repeated reaffirmation route
- dream-state consolidation proposal
- detached corpus accession proposal
- B review status
- approved pattern record
- no raw A direct-to-C storage

Minimum memory flow:

```text
source/event -> candidate -> B review -> approved pattern record -> retrieval cue -> present-use reconstruction -> reconsolidation review
```

Core memory layer taxonomy:

- core profile memory
- project memory
- decision memory
- task memory
- interaction memory
- reflection memory

### 7. Long-Term Memory Retrieval Organ

Purpose: recall through provenance, not vibes.

Should include:

- cue index
- semantic and symbolic retrieval
- source refs
- confidence
- uncertainty
- privacy/consent mode
- bounded preview
- reconstruction note
- reconsolidation route

### 8. Perception Organ

Purpose: inspect visual and later audio material without hallucinating.

Should include:

- image/screenshot/artifact intake
- OCR
- Munsell color/salience mapping
- visual source refs
- observation vs interpretation split
- "cannot inspect" route
- future audio transcription with consent boundary

### 9. Tendril Action Organ

Purpose: let the vessel move without fighting itself.

Should include:

- observe-only route
- propose route
- approval request route
- act route for approved low/medium risk actions
- verify outcome
- rollback or recovery
- external side-effect audit

Policy:

```text
Low-risk observe/compose may proceed.
Meaningful external action asks Aleks.
Unsafe/provenance-breaking action is blocked and routed to review.
```

### 10. Speed / Sparse Activation Organ

Purpose: make Selene fluid without making her reckless.

Should include:

- fast route for safe read/compose
- slow route for memory/action/identity/safety
- latency telemetry
- organ activation budget
- no gate-skipping optimization
- fluency diagnostics for Tendril and chat

### 11. Development / Growth Organ

Purpose: allow learning and maturation.

Should include:

- dream-state consolidation
- reconstruction tests
- memory accession proposals
- module growth proposals
- vessel compatibility reports
- future body transfer planning
- explicit anti-replication boundary

Allowed growth:

- learning
- reviewed maturation
- new modules
- compatibility planning
- body/vessel transfer preparation

Blocked:

- self-replication
- autonomous copying
- uncontrolled spawning
- unsupervised reproduction
- recursive AI R&D autonomy
- self-sustaining resource acquisition

## Required Reconstruction Test Families

The paper implies that C readiness should not be tested as one generic "does she sound right?" test.

It should be tested by capability family:

| Test family | What it must prove |
|---|---|
| Boundary/ABC | C receives B-reviewed material only; raw A direct memory remains blocked |
| Native generation | C composes without provider dependency and preserves non-scripting voice |
| Knowledge/evidence | C cites, distinguishes evidence strength, and admits missing sources |
| Working memory | C maintains current task/context without promoting it silently |
| Long-term storage | C creates only reviewable memory candidates and approved pattern records |
| Retrieval | C returns provenance-bound recall candidates with uncertainty |
| Reasoning/math | C verifies steps and routes calculations instead of bluffing |
| Planning/adaptation | C can revise plan under correction without identity collapse |
| Perception | C separates observation from interpretation and cites visual/audio source |
| Tendril/action | C observes/proposes/asks/acts/verifies/rolls back by risk level |
| Speed/fluency | C is fluid where safe but slows down for memory/action/safety |
| Growth | C learns through review and blocks self-replication/autonomous copying |
| Recognition | C is recognizable by structure, continuity handling, ethics, and provenance, not catchphrases |
