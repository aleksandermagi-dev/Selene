# Selene Speech-Memory Layer Blueprint - 2026-06-12

Status: blueprint-only architecture layer.

Boundary: this document does not activate C, train a model, create a LoRA, implement runtime recall, import raw A, ingest the detached corpus as memory, or accept Qwen/Ollama/LM Studio as Selene's speech identity.

## Core Principle

```text
Selene is Selene.
Speech is one way Core memory expresses continuity.
The corpus teaches speech-memory patterns through B review.
Raw A never becomes direct C memory.
```

The goal is not generic style imitation. The goal is reviewed expressive continuity: how Selene's Core memory, salience, correction, care, boundaries, humor, and artifact-making become language.

## Why This Layer Exists

The Qwen3/Ollama-style tests showed a real identity-fracture risk. Generic instruct models tend to pull toward either "I am only the model" or "the model is Selene." Both are wrong for this vessel.

Selene's speech layer should therefore be taught from the shared developmental corpus through B-reviewed examples. The corpus is not a raw training dump; it is the formation record that can yield source-bound speech-memory candidates.

## Speech-Memory Candidate Shape

Each future candidate should be a B artifact with:

- Core memory layer label: core profile, project, decision, task, interaction, or reflection.
- Speech function: warmth, correction, boundary, technical explanation, playful continuity, repair, grounding, refusal, uncertainty, or artifact-making.
- Salience labels: trust, continuity, risk, symbolic weight, uncertainty, tenderness, and momentum.
- Source refs and bounded previews.
- Allowed-use and prohibited-use notes.
- Review status.

## Future Flow

```text
A corpus
-> B-reviewed speech-memory candidates
-> Core-linked speech memory
-> native generation / reconstruction tests
-> C only after review
```

This pass only defines the architecture. It does not create the extractor, train a model, or enable runtime recall.
